Laravel Eloquent

Retrieving Models

  • all(): Retrieve all models
  • find($id): Find a model by its primary key
  • findOrFail($id): Find a model by its primary key or throw an exception
  • first(): Retrieve the first model
  • firstOrFail(): Retrieve the first model or throw an exception
  • get(): Retrieve all models matching the query
  • pluck($column): Retrieve a single column's value
  • chunk($count, $callback): Process models in chunks
  • where($column, $operator = null, $value = null, $boolean = 'and'): Add a basic where clause

    • Aggregates

  • count(): Get the count of models
  • max($column): Get the maximum value of a column
  • min($column): Get the minimum value of a column
  • sum($column): Get the sum of a column
  • avg($column): Get the average of a column

    • Inserting & Updating Models

  • create(array $attributes): Save a new model and return the instance
  • firstOrCreate(array $attributes, array $values = []): Get the first model matching the attributes or create it
  • updateOrCreate(array $attributes, array $values = []): Update or create a model matching the attributes
  • save(): Save the model to the database
  • update(array $attributes): Update the model in the database
  • increment($column, $amount = 1): Increment a column's value
  • decrement($column, $amount = 1): Decrement a column's value

    • Deleting Models

  • delete(): Delete the model from the database
  • destroy($ids): Destroy models by primary keys
  • forceDelete(): Permanently delete a model

    • Query Scopes

  • scopeName($query): Define a query scope

    • Relationships

  • hasOne(): Define a one-to-one relationship
  • belongsTo(): Define an inverse one-to-one or many relationship
  • hasMany(): Define a one-to-many relationship
  • belongsToMany(): Define a many-to-many relationship
  • morphTo(): Define an inverse polymorphic relationship
  • morphOne(): Define a one-to-one polymorphic relationship
  • morphMany(): Define a one-to-many polymorphic relationship
  • morphedByMany(): Define a many-to-many polymorphic relationship

    • Eager Loading

  • with($relations): Eager load relationships

    • Constraining Eager Loads

  • withCount($relations): Eager load relationships with count

    • Accessors & Mutators

  • getAttribute($key): Get an attribute from the model
  • setAttribute($key, $value): Set a given attribute on the model
  • getOriginal($key = null, $default = null): Get the original value of an attribute

    • Timestamps

  • touch(): Update the model's updated_at timestamp

    • Other Methods

  • toArray(): Convert the model instance to an array
  • toJson($options = 0): Convert the model instance to JSON
  • fresh($with = []): Reload the current model instance with fresh attributes
  • replicate(): Clone the current model instance into a new, non-existing instance

    • Where Methods

    Basic Where Clauses

  • where($column, $operator = null, $value = null, $boolean = 'and'): Basic where clause
  • orWhere($column, $operator = null, $value = null): Add an "or where" clause to the query

    • Where Clauses with Specific Comparisons

  • whereColumn($column1, $operator = null, $column2 = null, $boolean = 'and'): Compare two columns
  • orWhereColumn($column1, $operator = null, $column2 = null): Add an "or where" clause comparing two columns

    • Where Between Clauses

  • whereBetween($column, array $values, $boolean = 'and', $not = false): Where value is within a range
  • orWhereBetween($column, array $values): Add an "or where" clause within a range
  • whereNotBetween($column, array $values, $boolean = 'and'): Where value is not within a range
  • orWhereNotBetween($column, array $values): Add an "or where" clause not within a range

    • Where In Clauses

  • whereIn($column, $values, $boolean = 'and', $not = false): Where value is in a given array
  • orWhereIn($column, $values): Add an "or where" clause where value is in a given array
  • whereNotIn($column, $values, $boolean = 'and'): Where value is not in a given array
  • orWhereNotIn($column, $values): Add an "or where" clause where value is not in a given array

    • Where Null Clauses

  • whereNull($column, $boolean = 'and', $not = false): Where value is null
  • orWhereNull($column): Add an "or where" clause where value is null
  • whereNotNull($column, $boolean = 'and'): Where value is not null
  • orWhereNotNull($column): Add an "or where" clause where value is not null

    • Where Date Clauses

  • whereDate($column, $operator, $value, $boolean = 'and'): Where a date is matched
  • orWhereDate($column, $operator, $value): Add an "or where" clause for dates

    • Where Time Clauses

  • whereTime($column, $operator, $value, $boolean = 'and'): Where a time is matched
  • orWhereTime($column, $operator, $value): Add an "or where" clause for times

    • Where Day, Month, and Year Clauses

  • whereDay($column, $operator, $value, $boolean = 'and'): Where a day is matched
  • orWhereDay($column, $operator, $value): Add an "or where" clause for days
  • whereMonth($column, $operator, $value, $boolean = 'and'): Where a month is matched
  • orWhereMonth($column, $operator, $value): Add an "or where" clause for months
  • whereYear($column, $operator, $value, $boolean = 'and'): Where a year is matched
  • orWhereYear($column, $operator, $value): Add an "or where" clause for years

    • Where JSON Clauses

  • whereJsonContains($column, $value, $boolean = 'and'): Where a JSON array contains a value
  • orWhereJsonContains($column, $value): Add an "or where" clause for JSON array containing a value
  • whereJsonLength($column, $operator, $value, $boolean = 'and'): Where the length of a JSON array is matched
  • orWhereJsonLength($column, $operator, $value): Add an "or where" clause for JSON array length

    • Raw SQL Clauses

  • whereRaw($sql, $bindings = [], $boolean = 'and'): Add a raw where clause
  • orWhereRaw($sql, $bindings = []): Add a raw "or where" clause

    • Nested Where Clauses

  • whereNested(Closure $callback, $boolean = 'and'): Add a nested where clause
  • orWhereNested(Closure $callback): Add a nested "or where" clause

    • Subquery Where Clauses

  • whereExists(Closure $callback, $boolean = 'and', $not = false): Where a subquery exists
  • orWhereExists(Closure $callback, $not = false): Add an "or where" clause for existence of a subquery
  • whereNotExists(Closure $callback, $boolean = 'and'): Where a subquery does not exist
  • orWhereNotExists(Closure $callback): Add an "or where" clause for non-existence of a subquery
